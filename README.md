# Cyclistic Bike Share Case Study
Bike share data analysis using SQL and Tableau

## Project Overview
This project analyzes Cyclistic bike-share data to understand how casual riders and annual members use bikes differently. The goal is to identify patterns that can help design marketing strategies to convert casual riders into annual members.

---

## Business Problem
How do casual riders and annual members use Cyclistic bikes differently, and how can these insights be used to increase annual memberships?

---

## Data Source
- 12 months of Cyclistic trip data (Divvy system, Chicago)
- Public dataset
- No personal user data included (privacy protected)

---

## Tools Used
- SQL (Google BigQuery)
- Tableau (data visualization)
- Excel (initial exploration)

---

## Data Cleaning Steps
- Combined 12 monthly datasets
- Removed null and duplicate values
- Filtered invalid ride durations (negative or extreme values)
- Created new fields:
  - ride length
  - day of week
  - month

---

## Key Insights

### 1. Ride Duration
- Casual riders take longer trips
- Members take shorter, more efficient trips

### 2. Weekly Patterns
- Casual riders peak on weekends
- Members ride mostly on weekdays

### 3. Seasonal Trends
- Casual usage increases in summer
- Member usage is consistent year-round

---

## Conclusion
Casual riders primarily use Cyclistic for leisure, while members use it for commuting. This presents a strong opportunity to convert casual users by highlighting membership value for frequent riders.

---

## Recommendations
1. Target weekend casual riders with membership promotions  
2. Highlight cost savings of annual membership  
3. Use digital marketing to reach frequent casual users
